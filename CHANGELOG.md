# Portfolio Website — Build & Change Log

A record of everything done to `rajpcrj.github.io` — the futuristic redesign,
the project restructure, the new projects, and all the fixes along the way.

Live site: **https://rajpcrj.github.io**

---

## 1. Futuristic dark redesign

Completely rewrote the look into a black / dark, futuristic theme.

- **`style.css`** — full rewrite driving every page:
  - Deep black base (`#05060a`) with a drifting multi-color nebula glow
    (cyan / violet / magenta) and a faint perspective tech grid that fades out.
  - Glassmorphism panels (frosted, blurred sections) with a glowing cyan→violet
    top edge.
  - Gradient neon page title that pulses; monospace `// SOFTWARE · ROBOTICS · AI`
    eyebrow; pill-shaped contact bar.
  - Responsive project-card grid: hover lift, neon-cyan glow, cursor-tracking
    spotlight, images that go desaturated → full color + zoom on hover.
  - Neon scrollbar, glowing text selection, reduced-motion + mobile handling.
- **`script.js`** — kept the data loading, added:
  - Animated **particle constellation** canvas (nodes link to each other and reach
    toward the mouse).
  - **Scroll-reveal** animations as sections/cards enter the viewport.
  - Project-card hover spotlight tracking.

## 2. Projects restructured into per-folder layout

Each project moved from a flat `.html` file into its own folder served as
`index.html`, so the URL is a clean folder path and each project can hold its own
assets (videos, graphs, etc.).

```
projects/
├── project1/   index.html (+ demo.mp4)   — Home Safety System
├── project3/   index.html (+ demo.mp4)   — Sprout
├── go2/        index.html (+ demo.mp4)   — Go2 RL Locomotion
└── shm_fanout/ index.html (+ graphs/)    — ros2_shm_fanout
```

- Fixed all relative paths inside the moved files (`../style.css` →
  `../../style.css`, same for `script.js` and the back link).
- Added a glowing **"← Back to portfolio"** pill to every project page.
- **Project links point at `…/index.html` explicitly** so they work over `file://`
  (a bare folder URL fails when opened locally, only a web server auto-serves the
  folder's index).

## 4. Auto-numbering + manual ordering

- Project card numbers (**"Project 1:"**, **"Project 2:"**, …) are **generated by
  JavaScript from card position** — no hardcoded numbers in the HTML titles.
- **`project_index_list.json`** — a manually-editable file mapping each project to
  an `order` number:
  - Lower number = shown first.
  - Projects sharing the same number are sorted **alphabetically by title**.
  - `id` must match the project folder name under `projects/`.
- `script.js` reorders the cards from this file (with an embedded fallback so it
  also works on `file://`), then renumbers them to match the new order.
