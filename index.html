Home Safety System – Fall Detection & Smart Stove Control

A wearable safety system designed to protect elderly individuals living alone. Using a dual-axis accelerometer and two BS2 microcontrollers communicating over Bluetooth, the device detects falls in real time and triggers a 30-second cancel window before taking action. If no cancellation is received, it automatically texts the user's GPS location to a pre-set emergency contact and simultaneously shuts off the electric stove via a Solid State Relay — preventing hazards the user may be unable to address themselves. The system also includes a soft-reset button, a kill switch, a buzzer, and an LED for user feedback. Prototype cost was ~$404; mass production cost analysis brought this down to ~$96.54 with component substitutions and bulk ordering.

Sprout – Autonomous Greenhouse Plant Care System

Sprout is an autonomous mechatronic system developed for ROB-GY 6103: Advanced Mechatronics at NYU Tandon School of Engineering. It automates the care of greenhouse plants through a multi-bot architecture driven by real-time soil sensor data. A water bot navigates a 3×2 plant grid using IR line-following, reads soil moisture via capacitive sensors connected to an Arduino, and dispenses precise amounts of water through a solenoid valve based on soil profiles built from gravimetric analysis. A second NPK bot autonomously measures nitrogen, phosphorus, and potassium levels using an RS-485 Modbus sensor, then dispenses compost via a servo-actuated mechanism. A Raspberry Pi camera bot captures plant images served through a Flask web server with AI-based plant identification. Inter-device communication is handled via ESP-NOW protocol between ESP32 modules, enabling fully wireless, autonomous coordination across all three bots with no manual intervention.

Go2 RL Locomotion – Training a Unitree Quadruped to Walk in Isaac Lab

A deep reinforcement learning project (ROB-GY 6323, NYU) that trains a Unitree Go2 quadruped to walk inside NVIDIA Isaac Lab. Using PPO across thousands of GPU-parallel environments on NYU's Greene HPC cluster, the policy is shaped through principled reward terms — velocity tracking, posture stabilization, foot clearance, slip minimization, and smooth actions — and made robust through domain randomization with randomized masses, friction, and external push disturbances for sim-to-real transfer. The full pipeline runs on SLURM, producing TensorBoard metrics, model checkpoints, and rollout videos of the trained gait.

ros2_shm_fanout – A Shared-Memory Transport for High-Fan-Out Perception

While running a multi-camera perception stack in NVIDIA Isaac Sim, ROS 2's default transport became the bottleneck: with N camera streams and K subscribers, every frame was serialized and copied per consumer, so load scaled as N×K — pinning the CPU and stalling the simulator itself. I built a same-host shared-memory transport that writes one copy of each frame into /dev/shm and signals readers with a single futex wake; every consumer maps the same physical pages and reads in place. Benchmarked against FastDDS, CycloneDDS+iceoryx, and ros2_shm_msgs using Apex.AI's performance_test methodology, it delivered ~8× less CPU per consumer, flat RAM regardless of subscriber count, and was the only transport to hold ~30 fps with zero frame loss out to 64 consumers. It's type-agnostic (all ~330 ROS message types) and ROS-agnostic — any process that can mmap a file can read the stream, no ROS required.
